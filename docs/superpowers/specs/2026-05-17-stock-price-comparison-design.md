# 单公司财务柱状图股价对比设计

## 背景

当前页面支持在单公司视图中把财务指标切换为柱状图，但柱状图只能展示财务指标本身。用户希望在阅读单家公司长期营收或净利润趋势时，额外叠加一条股价曲线，帮助读者观察“基本面增长”和“市场定价变化”之间是否出现背离，从而更容易发现企业的相对性价比。

## 目标

第一版在满足以下条件时提供“股价对比”能力：

1. 当前仅显示 1 家公司；
2. 当前图表模式为柱状图；
3. 当前指标为营收或净利润。

打开开关后，在同一张图中同时展示：

- 左轴：营收或净利润柱状图；
- 右轴：该公司每日复权收盘价曲线（USD）。

第一版目标是把功能做得克制、稳定、可解释，而不是把页面扩展成完整行情终端。

## 交互设计

### 开关显示规则

新增一个“股价对比”开关，放在现有“单公司视图”控制区附近，和“折线 / 柱状”切换形成同一组上下文控制。

开关只在以下条件同时成立时显示：

- `visibleCompanies.size === 1`
- `chartMode === "bar"`
- `metric in ["revenue", "netIncome"]`

### 状态规则

- 默认关闭；
- 若用户打开后再切换到多公司、折线图，或切换到毛利率 / P/E / ROE / 营收增速，开关自动隐藏并重置为关闭；
- 当用户重新回到合法场景时，开关仍保持关闭，避免跨上下文残留造成误解。

### 视觉表现

开关打开后：

- 财务指标保持柱状图，继续使用公司主题色；
- 股价以独立折线叠加，使用与柱子区分明显的辅助色；
- 左侧 Y 轴保留原有财务指标标题；
- 右侧新增 Y 轴，标题为 `股价（USD）`；
- 图例明确区分：
  - 当前财务指标；
  - 股价。

## 图表行为

### 图层职责

- 柱状图始终表达财务指标；
- 股价线仅作为辅助阅读层，不改变原有财务柱状图的数据逻辑和缩放逻辑；
- 左右双轴相互独立，避免营收/净利润数量级和股价数量级相互挤压。

### 时间与横轴

第一版保留当前 Chart.js 分类轴体系，不把现有图表整体改造成真正的时间轴。

实现原则：

- 财务柱状图继续按当前时间粒度展示：季度视图按季度、年度视图按年度；
- 股价线使用每日复权收盘价；
- 股价线在当前财务时间窗口内裁切，并连续铺展在对应窗口之间；
- 底部时间区间滑块仍以财务标签窗口为准，股价线跟随当前窗口同步裁切。

这样可以在不推翻现有季度 / 年度 / 滚动年化逻辑的前提下，先得到接近参考图的阅读效果。

### Tooltip

- 悬停柱状图时，展示当前财务周期及财务指标值；
- 悬停股价线时，展示交易日期与股价值；
- 若同一悬停位置同时存在两类数据，tooltip 同时展示对应财务指标和股价；
- 财务指标 tooltip 继续沿用现有单位格式；股价统一显示为美元。

### 导出

- PNG 导出继续基于当前图表状态；
- 若股价对比开关开启，导出的 PNG 必须包含右轴、股价线和图例。

## 数据设计

### 文件边界

新增独立行情数据文件：

- `price-data.js`

继续保留现有职责划分：

- `data.js`：财务数据；
- `price-data.js`：行情数据。

这样可以避免把“财报事实”和“市场行情”混入同一个数据文件，降低后续维护成本。

### 数据结构

建议结构：

```js
window.STOCK_PRICE_SOURCE_DATA = {
  meta: {
    generatedAt: "2026-05-17T00:00:00.000Z",
    source: "Yahoo Finance chart API",
    priceType: "adjusted-close"
  },
  companies: {
    nvidia: {
      daily: {
        "2016-07-01": 1.18,
        "2016-07-05": 1.2
      }
    }
  }
};
```

### 口径

第一版使用 **每日复权收盘价**，原因是：

- 长周期比较时，复权口径可以避免拆股和分红让历史价格失真；
- 当前功能关注长期定价变化，而不是盘中交易；
- 单公司阅读场景下，真实美元价格比归一化指数更直观。

## 刷新与维护

新增独立刷新脚本：

- `scripts/auto-refresh-price-data.mjs`

职责：

1. 拉取目标公司的每日复权收盘价；
2. 标准化为 `YYYY-MM-DD -> adjustedClose`；
3. 写入 `price-data.js`；
4. 记录 `generatedAt`、`source`、`priceType` 等元数据。

第一版保持与财务刷新脚本分离，而不是立刻并入 `scripts/auto-refresh-data.mjs`，原因是：

- 两类数据的来源、频率、失败模式不同；
- 独立脚本更容易排错；
- 后续若要接入 GitHub Actions，也能独立控制刷新频率。

第一版行情来源采用无需密钥的 Yahoo Finance chart 端点；前端只依赖本地 `price-data.js`，不直接耦合具体供应商。

## 错误处理与降级

- 若某家公司暂无股价数据，或当前窗口内没有有效股价点：
  - 开关仍可按条件显示；
  - 用户打开后给出轻量状态提示；
  - 图表继续正常显示财务柱状图，不阻断主功能。
- 若行情数据加载失败：
  - 页面继续正常渲染；
  - 开关开启后显示明确提示，不抛出未处理异常。

## 第一版不做

为了保持范围清晰，第一版暂不实现：

- 多公司股价叠加；
- P/E、ROE、毛利率、营收增速与股价联动；
- 多条行情线或指数基准；
- 盘中价格；
- 前复权 / 后复权切换；
- 归一化股价指数；
- 估值带、回撤、涨跌幅副图；
- 移动端专门布局重构；
- 将整张图全面升级为真正的时间轴图表。

## 验收标准

1. 仅在“单公司 + 柱状图 + 营收/净利润”场景显示股价对比开关；
2. 开关默认关闭，离开合法场景后自动关闭并隐藏；
3. 打开后，图中出现股价日线、右侧股价轴和清晰图例；
4. 营收 / 净利润柱状图、右侧股价线和 tooltip 均显示正确；
5. 时间区间滑块裁切后，财务柱状图和股价线同步限制在同一窗口；
6. 数据缺失或加载失败时，财务图主功能仍可用；
7. 导出 PNG 能完整保留开启股价对比后的图表状态。
