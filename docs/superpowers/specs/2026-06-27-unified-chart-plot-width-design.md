# 统一单公司图表绘图区宽度设计

## 背景

单公司视图目前有三种常用状态：折线图、柱状图，以及开启股价对比后的柱状图。三者使用同一个画布，但 Chart.js 会根据右侧价格轴是否参与布局，动态调整网格绘图区。结果是切换模式时，网格右边界发生移动；左侧主轴的动态宽度也会让这种差异更显眼。

用户希望统一的是网格绘图区的左右边界，而不是外层图表容器。股价刻度和标题仍放在网格右侧。

## 目标

三种单公司状态共享相同的横向绘图区：

1. 单公司折线图；
2. 单公司柱状图；
3. 单公司柱状图并开启股价对比。

切换状态后，横向网格线的起点、终点以及底部区间滑块的对齐位置保持不变。图表高度、数据范围、刻度算法、柱宽、折线样式和导出逻辑不在本次调整范围内。

## 方案

采用共享固定轴槽。单公司图表始终为主 Y 轴和右侧辅助轴保留同一组水平空间。未开启股价对比时，右侧轴槽继续参与布局，但隐藏刻度、标题和边框；开启后只显示这些内容，不再改变轴槽宽度。

主轴也使用统一的保留宽度，避免折线和柱状模式因刻度文本测量结果不同而改变左边界。现有外层右侧 padding 保持不变，用于容纳单公司折线的末端标签。

多公司视图不显示股价对比，不属于三种对齐状态。本次不会扩大它的右侧留白。

## 代码边界

在 `price-comparison.js` 增加一个纯布局函数，根据可见公司数量返回主轴和辅助轴的预留宽度。单公司返回固定的左右轴槽，多公司保留现有布局。`script.js` 的初次构建和后续刷新都调用同一函数，避免两条渲染路径出现不同结果。

Chart.js 配置遵循以下规则：

- 单公司模式下，`y` 和 `yPrice` 的 `afterFit` 使用共享预留宽度；
- `yPrice` 在单公司模式下参与布局；
- 只有存在股价数据集时才显示 `yPrice` 的刻度和标题；
- 右轴网格不绘制到主绘图区；
- 区间滑块继续读取 `chart.chartArea`，无需单独计算偏移。

## 降级与边界情况

如果股价数据缺失，右侧轴槽仍保持占位，但不会显示空刻度或空标题。窗口尺寸变化时，固定的是轴槽宽度，中央绘图区继续响应式缩放。窄屏沿用现有移动端画布和横向布局规则，不新增断点。

## 测试

先为纯布局函数增加失败测试，覆盖单公司折线、单公司柱状、开启股价对比的柱状图，以及多公司状态。实现后运行现有 Node 测试集。

浏览器验收在同一视口、同一公司、同一指标和同一时间范围内进行。分别记录三种状态的 `chartArea.left` 与 `chartArea.right`，两项都必须一致；同时确认右轴只在股价对比开启时可见，底部滑块仍与绘图区边界对齐，并检查控制台没有新增错误。

## 验收标准

1. 三种单公司状态的网格左右边界一致；
2. 未开启股价对比时不显示右轴刻度或标题；
3. 开启股价对比后，右轴内容完整显示且不挤压网格；
4. 区间滑块与网格边界保持同步；
5. PNG 导出保留当前模式的完整图表；
6. 现有自动化测试全部通过。
