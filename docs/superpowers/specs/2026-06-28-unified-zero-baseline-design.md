# 单公司图表零线统一设计

## 背景

单公司折线图和柱状图目前使用不同的纵轴范围算法。柱状图以零为基准，并为净利润中的微小负值保留一小段空间；折线图则按真实最小值、最大值和 padding 重新取整。同一组数据在两种模式间切换时，纵坐标 0 的像素位置会发生移动，纵轴刻度也随之跳动。

以英伟达净利润为例，历史最小值约为 -0.20B，最大值约为 58.32B。柱状图把下界压缩到最大值的 -2.5% 左右，折线图会按完整跨度向下取整，两种计算产生了明显不同的零线位置。

## 目标

单公司折线图、柱状图和开启股价对比后的柱状图共用相同的左侧纵轴范围。切换模式后，纵坐标 0、上下界和刻度位置保持不变。数据、横轴、绘图区宽度、柱宽和折线样式不变。

## 方案

单公司视图统一使用现有柱状图纵轴算法。这个算法适合作为共享基准，因为柱状图必须从零线起绘制，同时已经处理了负值：

- 营收等全正指标从 0 开始；
- 净利润中的微小负值使用紧凑下界，避免大段空白；
- 显著亏损仍使用完整负值范围，不裁切数据；
- 毛利率、P/E、ROE 和营收增速继续按现有柱状图规则确定范围。

折线图只复用纵轴范围，不改变数据点、曲线、tooltip 或标签。多公司视图没有柱状切换，继续使用原来的折线范围算法。

## 代码边界

在 `price-comparison.js` 增加纯函数 `getYAxisBoundsMode`。它根据可见公司数量和当前图表模式返回计算纵轴范围时应使用的模式：单公司固定返回 `bar`，多公司返回当前模式。

`script.js` 的 `computeYAxisBounds` 在进入现有分支前调用该函数。后面的范围计算保持原样，只把判断使用的模式换成解析后的范围模式。这样初次构建、刷新、主轴宽度测量和股价右轴零线对齐都会自然读取同一组左轴上下界，不需要分别修补。

## 测试

先为 `getYAxisBoundsMode` 写失败测试，覆盖单公司折线、单公司柱状和多公司折线。再增加脚本集成契约，确认 `computeYAxisBounds` 使用解析后的范围模式。

浏览器验收使用同一公司、时间范围和视口，依次切换折线、柱状和股价对比。重点检查净利润、全正指标与含显著负值的指标，确认左轴 0 线和刻度不移动、负值没有被裁掉，并检查股价右轴仍与左轴 0 线对齐。最后复验 PNG 导出和控制台。

## 验收标准

1. 单公司折线与柱状图的左轴最小值和最大值一致；
2. 三种单公司状态中，纵坐标 0 的像素位置一致；
3. 微小负值仍可见，显著负值不被裁切；
4. 多公司折线图范围不变；
5. 股价右轴继续与左轴零线对齐；
6. PNG 导出完整，自动化测试全部通过，控制台无新增错误。
